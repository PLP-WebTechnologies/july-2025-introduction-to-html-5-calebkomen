HTML5 Semantic Elements

Why Semantic HTML Matters

Semantic HTML provides meaning to web content, making it more accessible and improving SEO. Instead of using generic <div> tags, HTML5 offers elements that clearly define their purpose.

Common Semantic Tags

  • <header> - Introductory content or navigation
  • <nav> - Navigation links
  • <main> - Primary content of the page
  • <section> - Thematic grouping of content
  • <article> - Self-contained composition
  • <aside> - Tangentially related content
  • <footer> - Footer for its nearest sectioning content

Web Accessibility

Making Websites Usable for Everyone

Web accessibility ensures that people with disabilities can perceive, understand, navigate, and interact with websites.

Key Accessibility Practices

  • Use semantic HTML to provide meaning and structure
  • Provide text alternatives for non-text content
  • Ensure keyboard navigation works properly
  • Use sufficient color contrast
  • Make forms accessible with proper labels
  • Use ARIA attributes when necessary

Search Engine Optimization

On-Page SEO Techniques

Proper HTML structure helps search engines understand your content, which can improve your rankings.

SEO Best Practices

  • Use a clear, descriptive title tag
  • Include meta description with target keywords
  • Use heading tags in proper hierarchy (H1, H2, H3)
  • Optimize images with alt attributes
  • Create semantic URL structures
  • Ensure fast loading times
  • Make your site mobile-friendly
